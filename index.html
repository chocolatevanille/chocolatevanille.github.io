My Projects

Recipes

Knowledge Representation and Reasoning (KRR) AI in Python

Challenged with crafting a KRR AI solution from scratch for the course Knowledge Representation and Reasoning, I designed an AI ontology to provide ingredient substitutions (based around food preference and allergens) for a group of meal recipes, securing an A in the course.

I did this by first creating a scalable KRR database in Python, representing shared qualities of elements via genLs. Critically this program does not simply assign elements the qualities of their parent genL; instances inherit all appropriate qualities via backward chaining. Likewise the program employs forward chaining and reasoning to determine the qualities of ingredients in the recipes and provide appropriate substitutions.

Please note: I use "I" here despite this being a group project because I did all of the work. I do not mean most of the work. I mean literally, all of it. I envisioned the premise, wrote every line of code (as evidenced by GitHub history), recorded the demo, and wrote the final project report. If the team slacks, I am willing to put in the work.

GodmasterHelper

Object Oriented Programming

This executable allows players of the hit video game Hollow Knight to track their progress and abilities in the Godmaster DLC.

Using Python's class system, I assigned information to each boss in the DLC, such as name, pantheons, and some numbers data. From there, I designed a UI that would allow for user input and managed the details of how many times they died to each boss in a way that gave players info on what they needed to work on to improve. I made sure data could be stored in files and loaded as needed. Once base functionality was implemented, I designed a full GUI in Python complete with buttons, drop-down menus, images, pop-up windows, and more, with the goal of enhancing user experience, particularly in usability. Finally, I exported the program as an executable and published it on GitHub.

Portfolio Website

HTML, CSS, and JavaScript in Action

You're on it! Hope you like it!

Faced with the inadequacy of GitHub and a resume to describe myself to potential employers, I resolved to design a portfolio website. The primary concern was usability; no one is impressed by a non-functional or confusing website. Therefore, using a combination of CSS, HTML, and JS I set out to make it both attractive and highly functional, with an emphasis on mobile-friendly design using variable object sizes and transitioning the navigation bar to an expandable dropdown menu. That dropdown menu closes upon selecting an option, ensuring it doesn't overstay its welcome, and on desktop the current navbar element highlights to display where the user is on the webpage. I made sure to style unique and beautiful divs and buttons, including personal touches such as a unique, somewhat transparent wipe effect that masks a subtle but effective gradient flip on hover. For convenience, YouTube videos are embedded and smooth scrolling implemented. For scalability and potential changes down the road, I positioned elements in CSS and HTML with best practices, and imported icons from FontAwesome, a free resource for many identifiable images and logos.

r/PokemonExchange CSS

CSS on Reddit!

Reddit used to rely on moderators to style each individual subreddit, allowing for a creative and diverse array of forums. From 2016 to 2019 I styled the subreddit r/PokemonExchange, a medium sized community which at the time had several thousand viewers per month.

During this time I was also the head moderator of the community, so functionally I had creative control. My partner Dr. Roberto Medina and I had fun exploring the different avenues to individualize our community. We analyzed various elements of the CSS, and once comfortable, began our modifications. We injected CSS code to modify tables, links, headers, menus, lists, and sprites; customized colors; applied hover effects; smoothed borders; and designed and supported a fully functional dark mode. This was one of my first projects out in "the real world", and I'm very thankful for the experience. The CSS has had changes in banner and wiki images only since 2019, so our creation is still live! Accessing it requires the "old" prefix in place of "www" when navigating Reddit.
